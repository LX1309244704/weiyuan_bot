## 产品概述

基于现有微源 Bot 平台，实现项目驱动的群聊协作模式。用户在左侧导航栏选择项目后，中间栏展示数字员工列表和项目文件树，添加数字员工后自动进入群聊模式，支持@提及功能进行任务分配协作。

## 核心功能

### 1. 项目详情视图重构

- 点击项目后，中间栏改为上下布局
- 上半部分：数字员工列表 + 添加功能
- 下半部分：项目文件树展示

### 2. 自动群聊模式

- 添加第一个数字员工后，自动创建群聊会话
- 自动切换到聊天界面显示群聊
- 后续添加的员工自动加入群聊

### 3. @提及功能

- 输入框输入@符号显示成员下拉选择
- 消息中高亮显示@提及内容
- 支持点击@成员快速定位

### 4. 任务分配协作

- 数字员工可@其他员工分配子任务
- 主 Bot 自动拆解任务并分配
- 群成员协同完成任务

## 技术栈

- **框架**: React + TypeScript + Zustand 状态管理
- **样式**: Tailwind CSS
- **已有基础**: QQ风格4列布局、AgentGroupPanel任务协作、nanobot任务处理

## 实现方案

### 架构调整

1. **QQSessionList 重构**: 将项目详情视图改为上下布局，集成 FilePanel 的文件树组件
2. **自动群聊初始化**: 修改 `addBot` 逻辑，首次添加 Bot 时自动创建 `type: 'group'` 会话
3. **@提及系统**: 在 QQChatArea 实现提及下拉框、内容解析、消息高亮
4. **任务分配集成**: 复用现有的 `taskProcessor.ts`，与@功能联动

### 关键技术决策

- **状态管理**: 复用现有的 appStore，新增 `isGroupChatMode` 标记群聊状态
- **消息类型扩展**: Message 类型添加 `mentions` 字段记录@的 Bot ID 列表
- **文件树复用**: 提取 FilePanel 中的 FileTreeItem 组件供 QQSessionList 使用
- **任务分配**: 复用现有的 AgentGroupPanel 和 taskProcessor，通过@触发任务分配

### 性能考虑

- 文件树懒加载，避免一次性加载大量文件
- @下拉选择使用虚拟列表（如成员过多）
- 消息渲染使用 useMemo 避免重复解析@内容

### 自动群聊初始化逻辑

```typescript
// 在 addBot 中检测是否是项目的第一个Bot
addBot: (bot) => set((state) => {
  const isFirstBot = !state.bots.some(b => b.projectId === bot.projectId)
  const newState = { bots: [...state.bots, bot] }
  
  if (isFirstBot && bot.projectId) {
    // 自动创建群聊会话
    const groupSession = createGroupSession(bot.projectId, [bot.id])
    newState.sessions = [groupSession, ...state.sessions]
    newState.currentSessionId = groupSession.id
  }
  
  return newState
})
```
